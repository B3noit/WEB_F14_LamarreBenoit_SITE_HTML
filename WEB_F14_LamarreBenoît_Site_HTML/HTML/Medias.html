-LES IMAGES

-Les formats d'images.

Ils existe plusieurs formats d'image les "jpeg", JPEG est l’acronyme de Joint Photographic Experts Group c'est un format qui comporte 16 millions de couleurs différents.
Le format "png"(portable network graphics) est le format le plus récent, l’image peut être transparente sans être altérée.Le format "png" a été créé pour remplacer le format GIF, c'est un format sans perte spécialement adapté pour publier des images simples comprenant des aplats de couleurs.
La compression des "jpeg" est plus puissante donc on privilégie ces formats pour les images et photos.
Le format d’image animée, les "gif". Le Graphics Interchange Format, plus connu sous l'acronyme "GIF", est un format d'image numérique couramment utilisé sur le web Une caractéristique rare du GIF est le nombre de couleurs supportées : au maximum 256 car chaque pixel est représenté sur 8 bits, contrairement à des formats en 64 bits, qui eux peuvent afficher plus de 16 millions de couleurs.

-Afficher une image.

La balise permettant d'afficher une image est la balise "img" elle doit être accompagnée de l'attribut "src" qui lui indique la source de l'image.

Comme cela, l'attribut "alt" vous permet d'afficher un message si l'image n'arrive pas à charger, l'attribut "title" quand à lui permet d'afficher une infobulle quand la souris de l'utilisateur passe sur l'image.

-LES VIDEOS ET MUSIQUES

-La musique !

S’il y a encore quelques temps il était assez compliqué de lire un fichier audio ou mp3 depuis votre site web, le HTML5 à simplifié sa mise en place. A partir d’une simple balise à insérer dans votre page web, la lecture de votre fichier musicale s’effectuera simplement.
Pour utiliser ce lecteur HTML5, vous devez utiliser la balise <audio>. Cette balise a été implanté dans le HTML5 afin de faciliter la lecture de fichier musicaux ou sonore.

-Les vidéos.

La balise <video>, cousin de <audio> offre en HTML5 une solution simple, native pour les navigateurs pour l'intégration d'une vidéo dans une page web.

A nouveau, la compatibilité peut poser problème, donc il faut toujours fournir un maximum de sources différentes si vous souhaiter une compatibilité sur le plus grand nombre de navigateur possible.

L'attribut controls donne accès aux contrôles de lecture (boutons de navigation, volume, etc, selon les possibilités du navigateur), ou les masque s'il est omis.
L'attribut preload="auto" permet de de spécifier au navigateur de débuter le téléchargement de la vidéo tout de suite, en anticipant sur le fait que l'utilisateur lira la vidéo.
L'attribut autoplay="true" comme son nom l'indique, permet de lancer la lecture automatiquement. Cela peut également être problématique avec une connexion à faible bande passante ou sur un terminal mobile. Cependant il est préférable de ne pas imposer ses choix à l'utilisateur... et à sa connexion internet.
L'attribut poster="image.jpg" permet d'indiquer une image à afficher par défaut dans l'espace réservé par la vidéo, avant que la lecture de celle-ci ne soit lancée.
L'attribut loop indique que la lecture doit s'effectuer en boucle.
